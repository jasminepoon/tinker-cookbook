Foundations — start here

This page is about teaching a model to follow instructions and to prefer better answers. Three plain-math ideas make all of it work: a loss is one number measuring "how wrong," the sigmoid turns a score gap into a probability, and a loss mask picks which words actually get graded. Here they are from scratch.

What "loss" means, and why we take a log

Training needs a single number to push downward. For each next word, we take the probability the model gave the correct word and score it with . Confident and right costs almost nothing; confident and wrong costs a fortune. That asymmetry is the whole point.

The sigmoid: a score gap → a probability

A preference is yes/no, but a reward model outputs unbounded scores. The sigmoid squashes any number into , so "winner scored higher than loser" becomes "probability the winner is preferred." Only the gap matters, not the raw scores.

A loss mask: which words actually get graded

A training example contains both the user's prompt and the model's reply, but you only want the model graded on its own words. A mask is a row of 1s and 0s that switches the loss on for reply tokens and off for prompt tokens.

01

Instruction Tuning

Instruction fine-tuning (IFT / SFT) is the first post-training step: it takes a pretrained next-token model and teaches it to follow instructions in a conversational format. It is the foundation for all downstream preference tuning. It uses the same cross-entropy loss as pretraining — but only on the right tokens, in the right structure.

Chat-template renderer

Post-training data is stored model-agnostically as a list of {role, content} messages. Before tokenization, tokenizer.apply_chat_template(messages) renders it into one flat token sequence with special role markers. Edit the conversation and watch it render — special tokens are highlighted in rose. Try the different model families: each uses different markers, but the same structure.

Prompt masking & the loss mask

During IFT the model only learns to predict assistant responses — system and user tokens (and the role markers) are masked out of the loss. The model learns how to respond, not how to ask. Toggle the loss mask, and for multi-turn pick the masking strategy: all assistant turns (train every response) or final-turn only (mask all but the last). Grey tokens condition the prediction but contribute no gradient.

the cross-entropy loss — same as pretraining, but summed over a restricted set of tokens
only assistant tokens are in the loss mask; system / user / role-marker tokens are skipped
the model's predicted probability of the next token given all preceding tokens

IFT data: quality vs. quantity

"Data quality matters more than quantity." Early post-ChatGPT work (LIMA, no_robots) showed ~10K high-quality human samples could be state-of-the-art; current practice scales to ~1M+ synthetic prompts, but quality filtering is essential and returns diminish. This toy model lets you trade dataset size against per-sample quality: model skill rises with the log of effective clean data N·q, but noisy samples actively subtract. Find the sweet spot — bigger isn't always better.

Reward Models

A reward model compresses complex, subjective human judgments into a single scalar score. In RLHF it plays the role of the environment's reward signal — but unlike classical RL we get to learn it from human preferences. The canonical formulation is the Bradley–Terry model: humans compare two responses, and the model learns to score the preferred one higher.

Loss variants: BT, margin, ORM

The same contrastive idea has variants. Bradley–Terry is the base. Llama 2's margin loss subtracts a margin (from Likert gaps) inside the sigmoid, forcing larger score gaps for strongly-preferred pairs. Outcome RMs drop pairwise comparisons entirely: a per-token binary cross-entropy against a single correctness label broadcast over every completion token. Switch the loss and drag the inputs.

03

Rejection Sampling & Best-of-N

The simplest thing you can do with a reward model: generate many candidate completions, score them, keep the best, and (optionally) fine-tune on them. No policy gradients, no online RL — just filtered supervised learning. Best-of-N is the same procedure used at inference time without the fine-tuning step.

Best-of-N: expected best score vs. N

Sample N completions, score each with a (toy) reward model, keep the argmax: S(R)=argmaxj∈[1,N] rj. As N grows the expected best score rises — but with diminishing returns. The bars below are one fresh draw at your current temperature; the curve is the analytic expected max of N draws. Resample to see the variance.

In practice · Best-of-N still maximizes the same imperfect proxy reward model, just at inference time instead of via gradient steps — push N high enough and you'll surface the reward model's blind spots same as RL would, just with a softer failure curve.

Selection: top-per-prompt vs. top-overall

Given M prompts × N completions and their reward matrix R, two ways to pick training data. Top-per-prompt: one argmax per row → every prompt represented. Top-K overall: flatten and keep the global top K → optimizes absolute quality, but can give one easy prompt several completions and leave hard prompts with none.
