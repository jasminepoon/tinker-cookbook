01

The policy-gradient family

Every method in this lecture minimizes the same loss skeleton — a sum of log-probs weighted by an advantage −Σ logπθ(at|st)·Ψt. They differ only in what Ψt is (the baseline / advantage) and how updates are bounded (clipping). Climb the ladder and see exactly what each algorithm adds.

Algorithm ladder: REINFORCE → RLOO → PPO → GRPO

Click an algorithm to see its loss expression and which ingredients it switches on: a variance-reducing baseline, a learned value function with GAE for per-token credit, ratio clipping for a trust region, and group-relative advantages. Each rung keeps the previous gradient skeleton and changes one or two pieces.

Notice: the only thing that ever changes is Ψt (the advantage) and whether the ratio is clipped. PPO is the only rung that needs a value network.

The RLOO leave-one-out baseline

REINFORCE subtracts the batch-mean reward as a baseline. RLOO samples K completions per prompt and, for each one, subtracts the mean of its siblings: bi = (1/(K−1))Σj≠i Rj. A per-prompt baseline removes prompt-difficulty variance the batch mean leaves in. Drag the rewards and watch each advantage recompute.

Each advantage Ai = Ri − (mean of the other K−1). Unlike REINFORCE's single batch baseline, a hard prompt that scores low across the board still produces sensible centered advantages.

02

Proximal Policy Optimization

PPO buys lower variance and per-token credit assignment — at the cost of a learned value function, GAE, and more models in memory. Two pieces do the work: the clipped surrogate that keeps each update inside a trust region, and GAE that turns one terminal reward into per-token advantages.

PPO clipped surrogate

The importance ratio is ρ = πθ(a|s)/πold(a|s). PPO minimizes −min(ρ·A, clip(ρ,1−ε,1+ε)·A). In code that's torch.max(−A·ρ, −A·clamp(ρ)) — picking the more pessimistic gradient. Drag the ratio and watch the objective flatten the moment you leave [1−ε, 1+ε].

Generalized Advantage Estimation (GAE)

GAE turns the terminal reward into per-token advantages by propagating TD errors δt = rt + γV(st+1) − V(st) backward with decay (γλ): Ât = Σl(γλ)lδt+l. λ=0 is pure TD (low variance, high bias); λ=1 is Monte-Carlo (high variance, low bias). Drag γ and λ and watch per-token advantages reshape across the sequence.

03

GRPO & group-relative advantages

GRPO throws away the value function and GAE entirely. For each prompt it samples a group of G completions and computes advantages purely by comparing siblings: Âi = (Ri − μG)/σG. Same clipped loss as PPO, one fewer model in memory.

Group-relative advantages (the core GRPO idea)

Sample a group of G responses for one prompt, give them toy rewards, then center within the group: subtract the group mean and divide by the group std. The baseline is the group itself — better-than-average siblings get positive advantage, worse-than-average get negative. Watch the std-collapse warning when all rewards are equal (the NaN trap).

KL penalty placement: in-reward vs in-loss

Same goal — constrain drift from the reference policy — two placements. PPO folds −β·KLt into the per-token reward before GAE, so KL flows through advantage estimation. GRPO adds β·KLt as a separate loss term, leaving advantages untouched. Toggle and see what the per-token training signal becomes.

04

Memory budget & loss aggregation

Two practical realities. PPO juggles four models in memory; GRPO drops one. And the seemingly cosmetic choice of how you reduce per-token losses to a scalar changes training dynamics — biasing the model toward short or verbose answers.

The four models in memory (and how GRPO drops one)

PPO keeps the policy (trained), a learned value/critic, a frozen reference (KL anchor), and a frozen reward model resident at once. For a 7B model in fp16 that's ~14 GB each. Toggle to GRPO and the value network vanishes — no critic, no GAE, no value loss, ~one fewer copy. Pick a model size and see the total weight footprint.

Loss aggregation: per-sequence vs per-token vs fixed-length

Same gradient ∇logπ·A, three ways to reduce it to a batch scalar. Per-sequence (GRPO) gives each sequence equal weight → short sequences get bigger per-token gradients. Per-token (DAPO) gives each token equal weight → long sequences contribute more total gradient. Fixed-length (Dr. GRPO) divides by a constant Lmax. Edit the two toy sequences and compare.

Per-token gradient weight for a token in a sequence of length |a|: per-sequence = 1/(B|a|), per-token = 1/Σ|a|, fixed = 1/(B·Lmax). Equal columns mean equal per-token weighting.
